# ReMirage Protocols

本目录是 `docs/03-自研协议` 的首轮迁移结果，用来给协议域建立一个新的正式入口。

它不再延续旧目录中“单篇大文档同时承载定位、实现、配置、性能假设、协同矩阵”的写法，而是拆成三类内容：

1. `stack.md`
   定义 ReMirage 协议栈的分层、角色和协同关系。

2. `source-of-truth.md`
   定义每个协议或协议族的主真相源、运行时锚点和旧文档状态。

3. 单协议正式入口
   当前已建立 `gtunnel.md`、`gswitch.md`、`vpc.md`、`jitter-lite.md`、`npm.md` 与 `bdna.md`，用于沉淀仍然有效的核心协议语义。

4. `legacy-mapping.md`
   记录 `docs/03-自研协议` 旧文档各自迁往哪里。

## 这套目录回答什么

- ReMirage 协议域有哪些层次。
- 各协议分别负责什么，不负责什么。
- 某类协议事实应该去哪里看、去哪里改。
- 旧协议文档接下来如何逐步收敛。

## 不在这里重复定义什么

- protobuf 消息字段与编号。
- Gateway/Client 代码中的具体结构体、算法细节和实现分支。
- 仅对某次发布或整改有效的短期结论。

这些内容应继续由其各自的主真相源持有。

## 阅读顺序

1. 先看 `stack.md`，理解协议域整体结构。
2. 再看 `source-of-truth.md`，确认具体协议该看哪里。
3. 需要进入核心协议时，看 `gtunnel.md`、`gswitch.md`、`vpc.md`、`jitter-lite.md`、`npm.md` 与 `bdna.md`。
4. 需要处理旧资料时，再看 `legacy-mapping.md`。

## 当前迁移结论

首轮迁移已经明确了三件事：

- 控制与共享消息的主权归 `mirage-proto/*.proto`。
- 单协议运行时主权归各自实现目录，而不是旧协议文档。
- `docs/03-自研协议` 现阶段整体降级为迁移输入材料，不再继续承担长期主源职责。

第二轮迁移进一步完成了两件事：

- `G-Tunnel` 已有一份面向当前实现的正式语义入口。
- `G-Switch` 已有一份面向当前实现的正式语义入口。

第三轮迁移补上了一个关键缺口：

- `VPC` 已有一份面向当前实现的正式语义入口。

继续往下收之后，伪装扰动层已经多了一条正式主线：

- `Jitter-Lite` 已有一份面向当前实现的正式语义入口。

再往下收一轮之后，伪装扰动层的关键两条链路也补上了：

- `NPM` 已有一份面向当前实现的正式语义入口。
- `B-DNA` 已有一份面向当前实现的正式语义入口。
