#### 🚀 静默透明代理（2026 内核级拦截）

**战略目标**：从"手动配置"到"零侵入感知"

##### A. eBPF + TPROXY 透明拦截

**技术架构**：

```
业务流量（任意语言/框架）
    ↓
网卡层（NIC）
    ↓
eBPF 钩子（sockops + sk_msg）
    ↓
内核套接字层拦截
    ↓
TPROXY 透明重定向
    ↓
MirageFrame 本地监听端口
    ↓
G-Tunnel 加密隧道
```

**核心能力**：

| 能力 | 传统 iptables NAT | eBPF + TPROXY |
|------|------------------|---------------|
| 性能开销 | 15%-20% | < 3% |
| 原始连接信息 | 丢失 | 完整保留 |
| 用户态拷贝 | 2 次 | 0 次（零拷贝） |
| 业务代码修改 | 需要设置代理 | 零修改 |
| 支持协议 | TCP only | TCP + UDP + QUIC |

**兼容性保障（Fallback 机制）**：

```
启动检测：
├─ 检查内核版本（≥ 5.15）
├─ 检查 BTF 支持（eBPF CO-RE）
└─ 检查权限（CAP_NET_ADMIN + CAP_BPF）

方案选择：
├─ 条件满足 → eBPF 主方案（< 3% 开销）
└─ 条件不足 → TPROXY Fallback（15% 开销，保证业务不中断）
```

**云服务商兼容性**：

| 平台 | eBPF 支持 | Fallback 需求 |
|------|----------|--------------|
| AWS (Kernel 5.15+) | ✅ 完整支持 | 无需 |
| GCP (COS/Ubuntu) | ✅ 完整支持 | 无需 |
| 廉价 VPS (Kernel < 5.10) | ❌ 不支持 | 自动切换 TPROXY |
| 阿里云/腾讯云 | ⚠️ 部分支持 | 按需切换 |

**实现细节**：

```c
// eBPF 程序挂载点
BPF_PROG_TYPE_SOCK_OPS    // 套接字操作拦截
BPF_PROG_TYPE_SK_MSG      // 消息层重定向
BPF_PROG_TYPE_CGROUP_SOCK // cgroup 级别控制

// 关键系统调用
setsockopt(IP_TRANSPARENT)  // 保留原始目标 IP
splice()                    // 内核态零拷贝
```

**部署要求**：
- Linux Kernel ≥ 5.15（推荐）/ ≥ 4.19（Fallback）
- 需要 CAP_NET_ADMIN + CAP_BPF 权限
- 兼容 `cilium/ebpf` Go 库

##### B. 零拷贝性能优化 (Zero-Copy)

**传统流量路径（4 次拷贝）**：

```
[网卡] → [内核缓冲区] → [用户态程序] → [内核缓冲区] → [网卡]
   ↑          ↑              ↑              ↑          ↑
  DMA      拷贝 1          拷贝 2         拷贝 3      DMA
```

**零拷贝路径（0 次用户态拷贝）**：

```
[网卡] → [内核缓冲区] ←→ [Socket FD] → [网卡]
   ↑          ↑                           ↑
  DMA    splice() 直接拼接               DMA
```

**性能提升**：

| 场景 | 传统方案 | 零拷贝方案 | 提升 |
|------|---------|-----------|------|
| 1GB 文件传输 | 850ms | 320ms | 2.7x |
| CPU 占用 | 45% | 8% | 5.6x |
| 内存带宽 | 4GB/s | 0.5GB/s | 8x |

#### 🔐 G-Tunnel 物理特性强化（骨干网对抗）

##### A. 数据包重叠采样 (Overlap Sampling)

**对抗目标**：骨干网层面的流量统计分析（Traffic Analysis）

**实现机制**：

```
传统碎片化：
数据包 A: [1-100]  → 路径 1
数据包 B: [101-200] → 路径 2
数据包 C: [201-300] → 路径 3
⚠️ 风险：截获任意两条路径可通过包长度还原负载

重叠采样：
数据包 A: [1-120]   → 路径 1 (含 20 字节重叠)
数据包 B: [101-220] → 路径 2 (含 20 字节重叠)
数据包 C: [201-300] → 路径 3
✅ 防御：有效载荷隐藏在重合部分，单路径无法解析
```

**技术参数**：

| 参数 | 配置 |
|------|------|
| 重叠率 | 15%-25%（可配置） |
| 路径数 | 3-5 条（动态调整） |
| 冗余开销 | 额外 20%-30% 带宽 |
| 重组算法 | XOR 校验 + FEC 纠错 |

##### B. MTU 动态探测与诱饵包

**统一包尺寸策略**：

```
流程：
1. 启动时探测所有路径最小 MTU
2. 强制所有数据包 Padding 到该尺寸
3. 审查系统看到的流量：规格完全一致的"工业制砖"

效果：
├─ 无法通过包大小判断协议类型
├─ 无法通过长度分布识别应用特征
└─ 流量统计分析失效
```

**诱饵包注入**：

| 类型 | 占比 | 目的 |
|------|------|------|
| 空载包 | 5%-10% | 干扰包序列分析 |
| 重复包 | 3%-5% | 模拟网络抖动 |
| 乱序包 | 10%-15% | 破坏时序特征 |

```c
SEC("uprobe/libc:getaddrinfo")
int intercept_getaddrinfo(struct pt_regs *ctx) {
    char *hostname = (char *)PT_REGS_PARM1(ctx);
    
    // 1. 查询 M.C.C. 下发的 IP 映射表
    struct ip_mapping *mapping = bpf_map_lookup_elem(&dns_override_map, hostname);
    
    if (mapping) {
        // 2. 直接返回预置 IP（绕过 DNS）
        struct addrinfo *result = (struct addrinfo *)PT_REGS_PARM4(ctx);
        result->ai_addr = &mapping->ip;
        
        // 3. 跳过原始 DNS 查询
        bpf_override_return(ctx, 0);
    }
    
    return 0;
}

// BPF Map：存储域名 → IP 映射
struct {
    __uint(type, BPF_MAP_TYPE_HASH);
    __uint(max_entries, 10000);
    __type(key, char[256]);   // 域名
```

```go
    proposal := RaftProposal{
        Type:      "DOMAIN_SWITCH",
        OldDomain: m.activeDomain,
        NewDomain: newDomain,
        Timestamp: time.Now(),
    }
    
    // 3. 等待 Raft 集群确认（< 100ms）
    if err := m.raft.Propose(proposal); err != nil {
        return err
    }
    
    // 4. 推送到所有 MirageFrame 节点
    return m.broadcastToEdges(newDomain)
}

        mf.removeDomain(oldDomain)
```

```c
struct mimic_params {
    __u32 mean_iat_us;      // 平均包间隔（微秒）
    __u32 stddev_iat_us;    // 标准差
    __u32 burst_size;       // 突发包数量
    __u32 packet_size_min;  // 最小包长
    __u32 packet_size_max;  // 最大包长
};

struct {
    __uint(type, BPF_MAP_TYPE_HASH);
    __uint(max_entries, 16);
    __type(key, __u32);   // 模板 ID
    __type(value, struct mimic_params);
} mimic_config_map SEC(".maps");

// 内核层根据 M.C.C. 下发的 ID 实时调整发包节奏
SEC("tc")
int apply_mimicry(struct __sk_buff *skb) {
    __u32 template_id = load_template_id(skb);
    
    // 查询当前模板参数
    struct mimic_params *p = bpf_map_lookup_elem(&mimic_config_map, &template_id);
    if (!p) {
        return TC_ACT_OK; // 无拟态
    }
    
    // 计算延迟（高斯分布）
    __u64 delay_us = gaussian_sample(p->mean_iat_us, p->stddev_iat_us);
    
    // 设置 skb 时间戳（硬件级定时）
    skb->tstamp = bpf_ktime_get_ns() + (delay_us * 1000);
    
    // 调整包长（Padding）
    adjust_packet_size(skb, p->packet_size_min, p->packet_size_max);
    
    return TC_ACT_OK;
}
```

### 8.2 G-Tunnel：多路径重叠采样算法

#### A. 核心逻辑实现

```go
// 重叠采样分片算法
type OverlapSampler struct {
    ChunkSize   int     // 基础分片大小：400 字节
    OverlapSize int     // 重叠大小：100 字节
    Paths       []Path  // 可用路径列表
}

func (s *OverlapSampler) Split(data []byte) []Fragment {
    fragments := []Fragment{}
    offset := 0
    pathIndex := 0
    
    for offset < len(data) {
        // 计算当前分片范围（含重叠）
        start := offset
        end := min(offset + s.ChunkSize + s.OverlapSize, len(data))
        
        fragment := Fragment{
            Data:      data[start:end],
            Path:      s.Paths[pathIndex % len(s.Paths)],
            SeqNum:    pathIndex,
            OverlapID: generateOverlapID(start, end),
        }
        
        fragments = append(fragments, fragment)
        
        // 下一个分片起点（减去重叠部分）
        offset += s.ChunkSize
        pathIndex++
    }
    
    return fragments
}

// 接收端重组（去重 + 校验）
func (s *OverlapSampler) Reassemble(fragments []Fragment) []byte {
    // 1. 按 SeqNum 排序
    sort.Slice(fragments, func(i, j int) bool {
        return fragments[i].SeqNum < fragments[j].SeqNum
    })
    
    // 2. 利用重叠部分进行 XOR 校验
    result := []byte{}
    for i, frag := range fragments {
        if i == 0 {
            result = append(result, frag.Data[:s.ChunkSize]...)
        } else {
            // 校验重叠部分
            prev := fragments[i-1]
            if !verifyOverlap(prev, frag, s.OverlapSize) {
                // 触发 FEC 纠错
                frag = s.recoverWithFEC(prev, frag)
            }
            result = append(result, frag.Data[s.OverlapSize:]...)
        }
    }
    
    return result
}
```

```go
func verifyOverlapAVX512(prev, curr []byte, size int) bool {
    // 使用 AVX-512 指令集
    // 8 个 64 字节块并行 XOR
    for i := 0; i < size; i += 64 {

        v1 := _mm512_loadu_si512(&prev[len(prev)-size+i])
```

```go
        // 低强度拟态（性能优先）
        mf.mimicry.Strength = 0.3
        mf.mimicry.PhysicalNoise = false
        
    case THREAT_MEDIUM:
        // 中等强度拟态（平衡）
        mf.mimicry.Strength = 0.6
        mf.mimicry.PhysicalNoise = true
        
    case THREAT_HIGH:
        // 高强度拟态（安全优先）
        mf.mimicry.Strength = 1.0
        mf.mimicry.PhysicalNoise = true
        mf.mimicry.RouterQueue = true
        mf.mimicry.PacketReorder = true
    }
    
    // 推送到 eBPF 内核
    mf.ebpf.UpdateMimicryConfig(mf.mimicry)
}
```

#### 🚀 逃逸 (Escape)：我能去哪？

**战略目标**：基础设施瞬态化 + 热切换零感知

##### A. 热切换 (Hot Switch)

**问题**：切断连接的瞬间是最危险的

**解决方案**：多路径重叠采样 + 后台预握手

```go
// G-Switch 热切换流程
func (gs *GSwitch) hotSwitch(oldDomain, newDomain string) error {
    // 1. 后台预握手新节点（不中断现有连接）
    newConn := gs.backgroundHandshake(newDomain)
    
    // 2. 多路径重叠传输（同时使用新旧节点）
    gs.enableOverlapMode(oldConn, newConn, duration=30*time.Second)
    
    // 3. 逐步迁移流量（0% → 100%）
    gs.gradualMigration(oldConn, newConn, steps=10)
    
    // 4. 关闭旧连接
    time.AfterFunc(30*time.Second, func() {
        oldConn.Close()
    })
    
    return nil
}

// 后台预握手（不影响现有流量）
func (gs *GSwitch) backgroundHandshake(domain string) *Connection {
    // 1. DNS-less 解析
    ip := gs.dnsless.Resolve(domain)
    
    // 2. QUIC 0-RTT 连接
    conn := gs.quic.Connect(ip, use0RTT=true)
    
    // 3. 预热连接（发送心跳）
    conn.SendHeartbeat()
    
    return conn
}

// 逐步迁移流量（用户无感知）
func (gs *GSwitch) gradualMigration(old, new *Connection, steps int) {
    for i := 0; i <= steps; i++ {
        ratio := float64(i) / float64(steps) // 0.0 → 1.0
        
        // 调整流量分配比例
        gs.setTrafficRatio(old, 1.0-ratio)
        gs.setTrafficRatio(new, ratio)
        
        // 每步间隔 3 秒
        time.Sleep(3 * time.Second)
    }
}
```

**热切换性能**：

| 阶段 | 延迟增加 | 丢包率 | 用户感知 |
|------|---------|--------|---------|
| 预握手（0-5s） | 0ms | 0% | 无感知 |
| 重叠传输（5-35s） | < 10ms | 0% | 无感知 |
| 逐步迁移（35-65s） | < 5ms | 0% | 无感知 |
| 完成切换（65s+） | 0ms | 0% | 无感知 |

```go
// 在切换期间使用重叠采样
func (gs *GSwitch) overlapTransmission(data []byte, old, new *Connection) {
    // 1. 切割数据（含重叠）
    fragments := gs.splitWithOverlap(data, overlapRatio=0.2)
    
    // 2. 分配到新旧路径
    for i, frag := range fragments {
        if i%2 == 0 {
            old.Send(frag) // 旧路径
        } else {
            new.Send(frag) // 新路径
        }
    }
    
    // 3. 接收端自动重组（去重）
    // 即使旧路径被封锁，新路径也能完整传输
}
```
