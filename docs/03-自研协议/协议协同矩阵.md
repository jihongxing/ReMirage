### 1.2 协议矩阵

| 协议 | 维度 | 核心能力 | 防御目标 | 实现语言 | 性能开销 |
|------|------|---------|---------|---------|---------|
| **G-Tunnel** | 空间 | 多路径 + 重叠采样 + FEC + Orchestrator | 单一链路被切断 | Go 控制 + C 数据面 | 带宽 +20% |
| **NPM** | 形态 | 动态 Padding + 协议拟态 | 包长度特征识别 | C (eBPF XDP) | 带宽 +12% |
| **B-DNA** | 指纹 | JA4 克隆 + 参数伪装 | TLS/QUIC 握手识别 | C (eBPF TC) | 延迟 +10ms |
| **G-Switch** | 存活 | 秒级域名转生 + DNS-less | 基础设施封锁 | Go (M.C.C.) | 延迟 +5ms |
| **Jitter-Lite** | 时间 | 高斯抖动 + 拟态矩阵 | IAT 统计分析 | C (eBPF TC) | CPU +1% |
| **VPC** | 背景 | 物理噪音 + 威胁自适应 | 用户行为画像 | C (eBPF TC) | 带宽 +10% |
| **WebRTC** | 伪装 | DTLS+SCTP DataChannel | UDP 封锁环境 | Go (pion/webrtc) | 延迟 +15ms |
| **ICMP Tunnel** | 穿透 | Echo Request/Reply Payload | Captive Portal | C (eBPF TC) + Go | 带宽极低 |
| **DNS Tunnel** | 生存 | Base32 子域名 + TXT/CNAME | 全封锁环境 | Go (miekg/dns) | 带宽极低 |

### 5.3 自愈机制

```go
type SelfHealing struct {
    protocols map[string]Protocol
    monitor   *HealthMonitor
}

func (sh *SelfHealing) AutoRecover() {
    ticker := time.NewTicker(5 * time.Second)
    
    for range ticker.C {
        for name, protocol := range sh.protocols {
            // 1. 健康检查
            if !sh.monitor.IsHealthy(protocol) {
                // 2. 尝试重启
                if err := protocol.Restart(); err != nil {
                    // 3. 启用降级方案
                    sh.enableFallback(name)
                    
                    // 4. 告警
                    sh.alert(name, err)
                }
            }
        }
    }
}
```

## 八、监控指标

### 8.1 协议健康度

```go
type ProtocolHealth struct {
    Name       string
    Status     string  // healthy/degraded/failed
    Latency    time.Duration
    Throughput float64
    ErrorRate  float64
    LastCheck  time.Time
}

// 健康度评分
func (ph *ProtocolHealth) Score() float64 {
    score := 100.0
    
    // 延迟评分
    if ph.Latency > 100*time.Millisecond {
        score -= 20
    }
    
    // 吞吐评分
    if ph.Throughput < 10*1024*1024 { // < 10 Mbps
        score -= 30
    }
    
    // 错误率评分
    score -= ph.ErrorRate * 100
    
    return score
}
```

### 8.2 协同效率

```go
type SynergyMetrics struct {
    TotalLatency    time.Duration
    TotalBandwidth  float64
    TotalCPU        float64
    DefenseLevel    int
    UserExperience  float64
}

// 协同效率 = 防御强度 / 性能开销
func (sm *SynergyMetrics) Efficiency() float64 {
    performanceCost := (sm.TotalLatency.Seconds() * 100) +
                       (sm.TotalBandwidth * 10) +
                       (sm.TotalCPU * 5)
    
    return float64(sm.DefenseLevel) / performanceCost
}
```

## 九、总结

### 9.1 协议协同优势

1. **全维度覆盖**：空间、形态、指纹、存活、时间、背景
2. **自适应防御**：威胁等级自动调整协议强度
3. **性能优化**：eBPF 内核态 + 并行处理
4. **故障自愈**：多层降级方案 + 自动恢复
5. **零感知切换**：用户连接不中断

### 9.2 关键指标

| 指标 | 保守模式 | 平衡模式 | 激进模式 |
|------|---------|---------|---------|
| 延迟增加 | +10ms | +20ms | +40ms |
| 带宽开销 | +20% | +35% | +50% |
| CPU 占用 | < 5% | < 10% | < 20% |
| 防御强度 | 60/100 | 80/100 | 95/100 |
| 用户体验 | 优秀 | 良好 | 可接受 |

### 9.3 适用场景

- **保守模式**：日常办公、社交媒体、轻度浏览
- **平衡模式**：视频会议、在线教育、远程办公
- **激进模式**：敏感通信、高风险环境、生存级需求
