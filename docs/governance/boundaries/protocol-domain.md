# Protocol Domain Boundary

本文件定义 ReMirage 的协议域边界，是“什么属于协议、什么只是策略或实现细节”的权威入口。

## 协议域的职责

ReMirage 的协议域负责定义以下内容：

- 组件之间如何交换控制与状态语义。
- 链路接入、切换、承载、伪装、扰动、恢复等通信行为的约束。
- 协议之间如何协同，而不是互相重叠定义。
- 哪些事实应由 `.proto`、运行时代码或说明文档持有。

## 属于协议域的内容

| 类别 | 说明 |
|------|------|
| 控制与信令契约 | Gateway、OS、Client 之间的消息、命令、字段语义 |
| 传输协议 | G-Tunnel 及其多路径、FEC、重组、降级与回退承载 |
| 存活与切换协议 | G-Switch 及其域名转生、切换、回退协同 |
| 伪装与扰动协议 | NPM、B-DNA、Jitter-Lite、VPC 等在承载层上的行为约束 |
| 回退承载 | WebRTC、DNS Tunnel、ICMP Tunnel 等作为受限环境下的承载方式 |

## 不属于协议域的内容

| 非协议内容 | 说明 |
|------------|------|
| 纯产品包装话术 | 官网文案、市场叙事、宣传语言不属于协议定义 |
| 纯实现技巧 | 仅与某个函数、库、性能优化细节有关但不影响协议语义的内容 |
| 业务管理规则 | 计费、用户运营、后台流程等不应伪装成协议定义 |
| 临时整改动作 | 某次专项排查、短期修复任务本身不构成协议主源 |

## 协议域的三层主权

1. 控制与共享消息主权在 `mirage-proto/*.proto`。
2. 单协议运行时行为主权在对应实现目录。
3. 协议目录、边界、归属关系主权在 `docs/protocols/`。

## 命名与拆分规则

- 一个协议文档只回答一个层级的问题。
- 解释性文档不应重复定义字段、报文、命令结构。
- 若某协议同时涉及 OS、Gateway、Client，必须先在协议目录中定义拥有者，再分别落实现。

## 变更规则

- 新增协议前，先补 `docs/protocols/` 中的目录入口和真相源归属。
- 修改共享消息结构前，先改 `.proto`。
- 修改单协议运行时行为前，先确认其主实现路径是否已经登记在协议真相源地图中。
