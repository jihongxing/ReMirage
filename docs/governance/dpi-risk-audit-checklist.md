# DPI Risk Audit Checklist

本文件用于把 ReMirage 当前面向 DPI、流量统计识别与主动探测的风险，拆成一份可执行的审计清单。

它回答的是：

- 当前最可能从哪些检测面被识别
- 当前已经有哪些正式承认的防护
- 还缺哪些证据
- 下一步应先验证什么

它不回答：

- 当前版本是否已经“可抗新一代 DPI”
- 某项销售表述是否可以直接对外承诺

上述问题统一以 `capability-truth-source.md` 为准。

## 一、使用规则

1. 本清单只审计当前真相源已经正式承认的能力，不外推未登记能力。
2. 每个检测面至少要产出一份可复验的证据，而不是只给口头结论。
3. 若某项验证失败，应先回写主文档的“当前真实能力”，再决定是否调整销售叙事。
4. 本清单优先服务 `Phase 2: 隐匿与数据面证据闭环`，但其中的 `P0` 项可前置执行。

## 二、风险等级定义

| 风险等级 | 含义 | 处理要求 |
|----------|------|----------|
| `P0` | 一旦成立，就可能直接导致被动识别、主动确认或大规模封锁 | 优先建立实验与复现证据 |
| `P1` | 不一定单独致命，但会显著增加识别概率或放大其他风险 | 在 `P0` 闭环后立即验证 |
| `P2` | 更偏边界收敛、长期稳态优化或文案治理 | 纳入后续周期持续修正 |

## 三、DPI 风险审计清单

| 检测面 | 当前防护 | 证据缺口 | 优先验证动作 | 风险等级 |
|--------|----------|----------|--------------|----------|
| 握手与栈指纹：TCP SYN / TLS ClientHello / QUIC Initial / JA4 | `B-DNA` 已接管 TCP SYN 重写、TLS ClientHello / QUIC Initial 协同重写入口，且已有 JA4 捕获、上报与 reset 联动 | 当前正式文档明确承认：TLS / QUIC 仍依赖用户态协同，尚不是完整浏览器栈或完整 TLS/HTTP2/HTTP3 行为克隆；缺少与真实浏览器 / 常见 CDN 前景流量的抓包对比证据 | 1. 在同一网络条件下采集 Chrome / 常见 uTLS 配置 / ReMirage 的握手流量 2. 抽取 JA3/JA4、TLS 扩展顺序、QUIC 参数、TCP options 3. 产出差异报告，确认是否存在稳定唯一签名 | `P0` |
| 首包与启动阶段统计特征：前几包长度、方向、burst 结构 | `B-DNA` 提供握手画像入口，`NPM` 提供 XDP 层包长填充与尾部剥离，支持固定 MTU / 随机区间 / 简化高斯三种填充模式 | 当前没有“首 5 到 10 个包”的长度、方向、熵与 burst 结构对比实验；不知道启动阶段是否仍能被快速归类 | 1. 批量采集不少于 1000 次新建连接 2. 统计前 10 包长度、方向、到达间隔、上下行比例 3. 与目标拟态流量做分布对比并训练简单分类器做可分性测试 | `P0` |

| 文案与真实能力偏差导致的误判风险 | 当前治理体系已经要求：没有独立实验结果前，不得把“有扰动能力”写成“对抗 ML/DPI 已显著领先” | 若销售矩阵或对外说法继续写满，团队容易误以为 DPI 风险已经关闭，导致验证优先级被稀释 | 1. 逐项对照 `capability-truth-source.md` 与销售矩阵 2. 标记所有超过当前证据边界的表述 3. 在实验报告产出前统一降级文案 | `P1` |

## 四、建议执行顺序

### 第一批：先验证最可能被快速识别的面

1. 握手与栈指纹
2. 首包与启动阶段统计特征
3. 时序 / IAT / 节奏识别
4. 主动探测 / 重放 / 未授权探针确认

### 第二批：验证“切换与伪装”会不会反而暴露

1. 多承载降级 / 回升过程的外显特征
2. 应用层行为拟态缺口
3. 背景噪声与环境拟态真实性

### 第三批：验证恢复与高压态侧信道

1. 节点发现 / 共振恢复侧信道
2. 压力态下一致性：eBPF 与用户态协同是否在高压下失真
3. 文案与真实能力偏差导致的误判风险

## 五、建议证据产物

建议每个检测面至少沉淀以下其中两类证据：

1. `pcap / pcapng` 抓包证据
2. 特征提取脚本或 notebook
3. 分类 / 对比实验报告
4. 运行日志与状态切换日志
5. 回写到真相源或派生材料的结论

建议统一使用如下命名习惯：

- `docs/reports/dpi-audit-<surface>.md`
- `artifacts/dpi-audit/<surface>/*.pcapng`
- `artifacts/dpi-audit/<surface>/summary.csv`

## 六、完成判定

某个检测面只有同时满足以下条件，才允许从“高风险待证”降级为“已验证边界”：

1. 已完成至少一轮可复验实验
2. 已有原始证据文件
3. 已有明确结论与限制说明
4. 结论已经回写主文档或相关派生材料

若只完成功能实现、未产出独立实验结果，则该检测面风险等级维持不变。
