# Document Lifecycle Rules

本文件定义 ReMirage 文档的状态规则，用来区分“输入材料、正式真相源、临时材料、归档材料”。

## 文档状态

| 状态 | 含义 | 是否可直接作为决策依据 |
|------|------|------------------------|
| `authoritative` | 该问题域的权威入口 | 可以 |
| `derived` | 从权威入口派生出来的摘要、导读、样例或教程 | 不可单独使用 |
| `input` | 迁移过程中的旧资料、纪要、草案、整改清单 | 不可直接使用 |
| `temporary` | 只在某个发布周期、专项行动或短期任务中有效 | 仅在限定范围内可用 |
| `archived` | 历史记录，保留审计价值但不再指导当前实现 | 不可 |

## 默认归类规则

- `README` 默认是 `derived`，除非它被明确声明为某问题域主入口。
- `ADR` 默认是“决策依据文档”，它记录为什么这样定，但运行时真相可能仍在代码。
- 会议纪要、整改清单、Spec 拆解、方向确认默认是 `input` 或 `temporary`。
- 发布就绪索引、上线封版清单默认是 `temporary`。
- 样例配置、教程、官网文案默认是 `derived`。

## 升级为权威文档的条件

一份文档只有满足以下条件，才能从 `input` 或 `derived` 升级为 `authoritative`：

1. 它定义的问题域边界清晰。
2. 它在 `source-of-truth-map.md` 中被明确登记。
3. 它不与已有权威入口重叠。
4. 它的维护责任明确，且变更路径清晰。

## 失效与降级规则

出现以下情况时，文档应降级或归档：

1. 已有新的权威入口接管该问题域。
2. 文档只保留历史语境，已不适合继续指导实现。
3. 文档内容与运行时真相长期偏离，且不再值得修复。

## 建议执行方式

后续迁移旧文档时，建议在文档头部增加状态标记，例如：

```md
Status: input
Target Truth: docs/governance/boundaries/product-scope.md
```

或：

```md
Status: derived
Source of Truth: mirage-proto/mirage.proto
```

本次不批量修改旧文档，但后续迁移时应优先补这种标记。
