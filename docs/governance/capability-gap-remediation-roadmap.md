# North Star Implementation Plan

本文件是 `capability-truth-source.md` 的执行 companion。

它不重新定义能力真相，只回答一件事：

围绕 `Hyper-Resilient Overlay Network` 与 `Zero-Trust SD-WAN 隐蔽数据链路` 这两个北极星目标，当前版本应按什么阶段推进、每一阶段要完成哪些里程碑、以什么证据判定通过。

## 一、使用规则

1. 本实施计划只能从 `capability-truth-source.md` 推导，不得自行扩展新的能力域。
2. 若主文档中的“当前真实能力”发生变化，应先更新主文档，再回写本实施计划。
3. 本实施计划只允许安排：
   - 测试补齐
   - 验证闭环
   - 编排收口
   - 证据沉淀
   - 配置与部署基线固化
4. 本实施计划不用于新增功能立项，不得借机扩大产品边界。
5. 每一阶段完成后，必须同时回写：
   - `docs/governance/capability-truth-source.md`
   - 对应验证清单 / 审计材料 / 派生叙事文档

## 二、北极星推进目标

本轮实施计划只服务两个正式北极星命名：

1. `Hyper-Resilient Overlay Network`
2. `Zero-Trust SD-WAN 隐蔽数据链路`

本轮推进原则不是“先把词讲满”，而是按顺序把以下三件事做实：

1. 先让链路连续性与恢复能力形成真实闭环
2. 再让隐匿性、eBPF 参与度和运行痕迹形成证据边界
3. 最后让对外命名、销售叙事和当前真实能力完全对齐

## 三、阶段总览

|------|----------|------------|------------|--------------|

## 四、阶段实施拆解

### Phase 1: 链路连续性闭环

本阶段决定 ReMirage 能否真正朝 `Hyper-Resilient Overlay Network` 迈出第一步。

#### 里程碑 `M1`: 承载矩阵冻结

目标：

- 明确当前版本正式承诺哪些承载
- 明确哪些承载只是“已接线待闭环”
- 避免继续把设计入口写成稳定承载能力

涉及能力域：

- 多承载编排与降级

验收证据：

1. 当前正式承诺承载矩阵
2. `QUIC -> WSS` 当前支持边界说明
3. `WebRTC / ICMP / DNS` 的状态标注回写

#### 里程碑 `M2`: 自动降级/回升演练通过

目标：

- 至少打通一条发布级的自动降级与回升链路

涉及能力域：

- 多承载编排与降级

验收证据：

1. 端到端降级/回升演练脚本或测试文件
2. 降级与回升的运行日志
3. 功能验证清单中的对应回写
4. 必要时 CI job 记录

#### 里程碑 `M3`: 节点阵亡恢复演练通过

目标：

- 证明“发现器存在”已经升级为“节点阵亡后可恢复”

涉及能力域：

- 节点恢复与共振发现

验收证据：

1. 失效主节点 -> 发现新入口 -> 完成切换的演练脚本
2. 终端日志 / 事件记录
3. 一条受控演练报告
4. 恢复成功、超时、失败回退判定标准

#### 里程碑 `M4`: 业务连续性样板通过

目标：

- 至少选 1 到 2 条业务样板，证明“业务连续”而不只是“连接存在”

涉及能力域：

- 会话连续性与链路漂移

验收证据：

1. 长连接 TCP / WebSocket / 持续请求流样板之一的切换测试
2. 中断/不中断判定日志
3. “传输层切换”与“业务层无感”分层结论
4. 主文档状态回写

### Phase 2: 隐匿与数据面证据闭环

本阶段决定 ReMirage 能否从“有协议和 eBPF 组件”升级为“对隐匿与数据面边界有实证认知的系统”。

#### 里程碑 `M5`: 隐匿实验方案冻结

目标：

- 先统一实验口径，再决定允许对外讲到什么程度

涉及能力域：

- 流量整形与特征隐匿

验收证据：

1. 实验设计文档
2. 握手指纹、包长分布、时序分布、简单分类器的测试方案
3. 对外允许 / 不允许表述清单

#### 里程碑 `M6`: 最小实验结果产出

目标：

- 产出最小但可信的实验结果，而不是继续用概念补位

涉及能力域：

- 流量整形与特征隐匿

验收证据：

1. 抓包结果
2. 统计结果
3. 实验结论与限制说明
4. `docs/暗网基础设施防御力评价矩阵.md` 回写

#### 里程碑 `M7`: eBPF 覆盖边界与性能证据完成

目标：

- 统一 eBPF 的真实参与边界，避免把“深度参与”写成“全链路零拷贝”

涉及能力域：

- eBPF 深度参与的数据面与防护

验收证据：

1. eBPF 覆盖图文档
2. 关键路径最小 benchmark / 观测数据
3. 对外表述统一稿
4. 主文档与派生文档回写

### Phase 3: 可运营基线固化

本阶段决定 ReMirage 能否从“有强组件”升级为“有长期可运营的部署与治理基线”。

#### 里程碑 `M8`: 最小运行痕迹基线冻结

目标：

- 明确默认部署、加固部署、极限隐匿部署三种边界

涉及能力域：

- 反取证与最小运行痕迹

验收证据：

1. 部署等级说明
2. 基线检查清单
3. 与 runbook 一致的配置示例

#### 里程碑 `M9`: 准入控制联合演练通过

目标：

- 证明鉴权、配额、熔断、日志脱敏能在真实链路下协同工作

涉及能力域：

- 准入控制与防滥用

验收证据：

1. “非法接入 -> 拒绝 -> 日志脱敏 -> 配额不受损”联合演练记录
2. “合法接入 -> 配额耗尽 -> 目标用户熔断”联合演练记录
3. 长期 smoke / critical 验证入口
4. 功能验证清单回写

### Phase 4: 北极星升级判定

本阶段不新增能力域，只负责把前面三阶段的证据收敛成正式能力升级。

#### 里程碑 `M10`: 主文档状态回写

目标：

- 根据真实证据，而不是叙事强度，回写能力状态

验收证据：

1. `capability-truth-source.md` 中对应能力域状态升级或维持不变
2. 每项升级均有证据锚点链接

#### 里程碑 `M11`: 派生材料收敛

目标：

- 让销售矩阵、场景文档、对外叙事与当前真实能力完全对齐

验收证据：

1. `market-positioning-scenarios.md` 回写
2. `docs/暗网基础设施防御力评价矩阵.md` 回写
3. 不再出现超出主文档边界的满额承诺

**M11 完成记录（2026-04-25）**：派生材料收敛完成。Market_Positioning 三个商业场景已增加能力边界限定语，头部已增加对齐声明。Defense_Matrix 评分已增加限定标注（部分实现域标注为目标分值，已实现（限定表述）域增加限定说明），话术章节已移除绝对化表述。跨文档一致性检查七域全部通过，无不一致项。详见 `docs/reports/cross-document-consistency.md`。

#### 里程碑 `M12`: 发布级验证入口固定化

目标：

- 把本轮成果沉淀为稳定的 release / audit / regression 入口

验收证据：

1. 功能验证任务清单回写
2. release gate / verify script / CI job 固化
3. 下一轮发布可直接复用的验证入口

### Phase 3 出关标准

1. `M8` 与 `M9` 全部完成
2. 部署基线与联合演练可进入长期运维验证流程

### Phase 4 出关标准

1. `M10` 到 `M12` 全部完成
2. `capability-truth-source.md`、派生叙事文档、验证入口三者一致
3. 对外命名与当前真实能力不再冲突

**Phase 4 实际达成情况（2026-04-25）**：
1. M10-M12 全部完成 ✅
2. Capability_Truth_Source / 派生叙事文档 / 验证入口三者一致 ✅（跨文档一致性检查七域全部通过）

## 七、升级完成判定

一项能力只有同时满足以下条件，才允许从“部分实现”升级：

1. 主文档中的验收标准全部满足
2. 有至少一个稳定的验证入口
3. 有明确证据文件或运行日志
4. 对外文案已与主文档同步

若只完成代码接线、尚未补齐验证证据，则状态维持不变，不得提前升级。
