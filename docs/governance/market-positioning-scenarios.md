# Market Positioning Scenarios

本文件用于承接 ReMirage 的商业化场景叙事与交付包装。

它回答的是：

- 这套系统在市场上可以被解释成什么
- 对哪些高价值场景有吸引力
- 适合用什么交付形态进入客户系统

它不回答：

- 当前版本是否已经具备这些能力
- 某项表述是否已经达到可对外交付级别
- 某个比较结论是否已经有实验充分证明

上述问题统一以 `product-scope.md` 与 `capability-truth-source.md` 为准。

## 一、正式市场定位

ReMirage 的正式目标定位是：

- `Hyper-Resilient Overlay Network`
- `Zero-Trust SD-WAN 隐蔽数据链路`

“影子网络”是叙事标签，不是验收术语。

## 二、核心商业场景

### 1. 跨境高价值指令与资产调度链路

适用于对链路可恢复性、低漂移和低暴露有强约束的场景，例如跨区域指令调度、关键资产控制面、长期稳定的专用数据链路。

> **能力边界（与 Capability_Truth_Source 对齐）**：多承载编排当前支持 QUIC→WSS 降级/回升（代码级模拟验证），WebRTC/ICMP/DNS 路径为已接线待闭环。流量整形与特征隐匿当前为部分实现，缺少受控环境实验结论，不可按满额隐匿效果承诺。

### 2. 高价值节点与关键基础设施保护

适用于需要降低节点暴露面、提升抗阻断与恢复能力的场景，例如关键基础设施节点、关键对外服务入口、需要快速转移和恢复的高价值节点。

> **能力边界（与 Capability_Truth_Source 对齐）**：节点恢复当前为代码级模拟验证（已实现（限定表述）），尚无真实节点阵亡演练证据。反取证支持默认/加固两种部署等级，极限隐匿部署部分配置项待补齐。

### 3. 特殊网络环境下的企业业务连续性链路

适用于对跨区域办公、研发同步、业务系统互联稳定性要求很高，但无法接受单一承载被限制后整体失联的企业场景。

> **能力边界（与 Capability_Truth_Source 对齐）**：会话连续性当前为传输层切换验证通过（已实现（限定表述）），业务层无感需真实网络环境单独演练量化，不可按"所有 TCP 业务无感"承诺。

## 三、标准交付形态

### 1. Managed Gateway

Mirage 以托管式网关能力向客户交付，负责策略、编排、恢复与运行治理。

### 2. Sidecar

Mirage 以本地边车代理方式进入客户业务环境。客户尽量少改业务代码，流量经受控本地入口进入 Mirage 数据链路。

### 3. SDK / Lib

Mirage 以受控集成形态进入客户系统，只面向高级集成场景开放。该模式不是开放平台许可，不意味着任何第三方都可以脱离 Mirage 治理体系独立封装或转售。
