# Mirage-Gateway 融合网关（极简版）

## 项目定位

**Mirage-Gateway**：融合网关，集成 MirageFrame（拟态伪装）+ DarkLink（司法管辖区轮转）

- **任务**：流量伪装、指纹对抗、多路径传输、威胁检测
- **架构**：C (eBPF 数据面) + Go (控制面)
- **角色**：前线战斗单元

---

## 核心能力

### 六大自研协议

| 协议 | 维度 | 核心能力 | 实现语言 |
|------|------|---------|---------|
| **NPM** | 形态 | 流量伪装（Padding 填充） | C (eBPF XDP) |
| **B-DNA** | 指纹 | 行为识别与对抗（JA4 指纹伪装） | C (eBPF TC) |
| **VPC** | 背景 | 噪声注入（光缆抖动模拟） | C (eBPF TC) |
| **Jitter-Lite** | 时间 | 时域扰动（IAT 随机化） | C (eBPF TC) |
| **G-Tunnel** | 空间 | 多路径传输（FEC 纠错） | Go 控制 + C 数据面 |
| **G-Switch** | 存活 | 域名转生（5 秒平滑切换） | Go (M.C.C.) |

### 协议协同矩阵

| 场景 | 协议组合 | 效果 |
|------|---------|------|
| 日常浏览 | NPM + Jitter-Lite | 基础隐蔽 |
| 视频会议 | NPM + B-DNA + VPC | 深度伪装 |
| 高风险环境 | 全协议栈 | 极限隐蔽 |
| 弱网环境 | G-Tunnel + Jitter-Lite | 0 丢包恢复 |
| IP 封禁 | G-Switch | 5 秒转生 |

## 核心协议详解

### NPM（流量伪装协议）

**目标**：通过 Padding 填充，让流量包长度分布符合目标应用特征

**技术要点**：
- XDP 层零拷贝追加 Padding
- 动态填充比例（10%-30%）
- 成本透明化（$0.10/GB 业务流量，$0.05/GB 防御流量）

### B-DNA（行为识别协议）

**目标**：对抗 JA4 指纹识别，伪装为合法应用

**技术要点**：
- TCP 握手阶段修改 Window Size
- TLS ClientHello 字段随机化
- 拟态模板库（Zoom/Netflix/Steam）

### VPC（噪声注入协议）

**目标**：模拟光缆物理抖动，对抗时序分析

**技术要点**：
- 纳秒级精度定时器
- 高斯分布噪声注入
- 威胁检测（SYN/XMAS/NULL 扫描）

### Jitter-Lite（时域扰动协议）

**目标**：随机化包间隔时间（IAT），对抗流量分析

**技术要点**：
- 修改 skb->tstamp（内核时间戳）
- 动态抖动范围（0-50ms）
- 威胁等级自适应调整

### G-Tunnel（多路径传输协议）

**目标**：30% 丢包环境下实现 0 丢包恢复

**技术要点**：
- Reed-Solomon FEC 编码（8 数据 + 4 冗余）
- AVX-512 指令集加速
- 多路径并发传输（3 条路径）
- 路径调度策略（round-robin/lowest-rtt/redundant）

### G-Switch（域名转生协议）

**目标**：IP 封禁时 5 秒内平滑切换域名

**技术要点**：
- 温储备池（10-15 个待命域名）
- 后台预握手
- 多路径重叠传输
- 用户无感知切换

## 防御体系

### 五级防御等级

| 等级 | 触发条件 | 防御措施 | 性能影响 |
|------|---------|---------|---------|
| 0 | 无威胁 | 基础伪装 | 延迟 +5ms |
| 1 | 低风险 | 增强伪装 | 延迟 +10ms |
| 2 | 中风险 | 深度伪装 | 延迟 +20ms |
| 3 | 高风险 | 全协议栈 | 延迟 +50ms |
| 4 | 极限风险 | 紧急转生 | 延迟 +100ms |

### 威胁检测

**检测类型**：
- SYN 扫描（TCP Flags = SYN）
- XMAS 扫描（FIN+PSH+URG）
- NULL 扫描（所有 Flags = 0）
- JA4 指纹探测
- 主动探测（连接后立即断开）

**响应机制**：
- 实时上报 Mirage-OS
- 累积效应（威胁计数）
- 全局黑名单同步
- 自动防御升级

## 系统免疫反应流程

```
1. 感知（VPC Ingress）
   └─ 检测到 SYN 扫描

2. 上报（Ring Buffer → Go）
   └─ 威胁事件推送到 Mirage-OS

3. 学习（threat_intel 表）
   └─ 累积威胁计数，更新全局黑名单

4. 下令（SyncHeartbeat）
   └─ 下发新防御等级和黑名单

5. 执行（eBPF Map）
   └─ 内核态立即应用新策略

6. 计费（billing_logs）
   └─ 记录防御流量开销
```

## 编译与部署

### 编译要求

**C 数据面**：
- clang + BPF target
- 内核版本 ≥ 5.15（推荐）或 ≥ 4.19（降级）
- BTF 信息（-g 参数）

**Go 控制面**：
- Go 1.21+
- CGO_ENABLED=1
- cilium/ebpf 库

### 部署流程

1. 编译 eBPF 程序：`make build-bpf`
2. 编译 Go 程序：`make build`
3. 启动 Gateway：`./bin/mirage-gateway`
4. 验证加载：`bpftool prog list`

## 核心优势

### 技术优势

- **零拷贝性能**：XDP 层直接修改数据包，无需用户态拷贝
- **纳秒级精度**：内核态定时器，Go 无法实现
- **AVX-512 加速**：FEC 编码性能提升 10 倍
- **内核态熔断**：配额耗尽立即阻断，无需用户态判断

### 商业优势

- **成本透明**：实时显示流量和费用
- **按量计费**：用多少付多少
- **分级定价**：标准/白金/钻石蜂窝
- **欠费熔断**：防止恶意刷量

### 安全优势

- **全球免疫**：威胁情报实时同步
- **自动进化**：策略引擎持续学习
- **物理隐匿**：司法管辖区轮转
- **零信任架构**：蜂窝隔离，单点失陷不影响全局

Mirage-Gateway 是一个集成了六大自研协议的融合网关，通过 C (eBPF) 数据面和 Go 控制面的二元架构，实现了：

- **形态伪装**：NPM 流量填充
- **指纹对抗**：B-DNA 行为伪装
- **时域扰动**：Jitter-Lite IAT 随机化
- **噪声注入**：VPC 光缆抖动模拟
- **空间逃逸**：G-Tunnel 多路径传输
- **存活保障**：G-Switch 域名转生

在 30% 丢包环境下实现 0 丢包恢复，IP 封禁时 5 秒内平滑切换，威胁检测实时上报，全局黑名单秒级同步，成本透明化，按量计费。

这是一个自进化、自防御、自计费的去中心化网络协议栈。
