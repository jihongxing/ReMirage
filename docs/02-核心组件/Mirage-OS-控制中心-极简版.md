# Mirage-OS 控制中心（极简版）

## 项目定位

**Mirage-OS**：控制中心，集成 M.C.C.（全球域名流水线调度）+ Web 后台

- **任务**：全球域名流水线调度、蜂窝隔离算法、死因分析算法、Web 指挥中心
- **架构**：高性能微服务（Go/Rust）+ 前端看板
- **角色**：战区指挥部

## 核心服务

### 域名流水线服务

**自动采购流程**：
1. 检查温储备池（< 10 个时触发）
2. 从冷储备池激活域名
3. 检查冷储备池（< 20 个时触发）
4. 自动采购新域名（Monero + Tor）

**域名采购器**：
- 生成一次性身份
- 选择注册商（避开高风险）
- Monero 混币支付
- 通过 Tor 注册

### 蜂窝管理服务

**蜂窝隔离算法**：
- 资源池独立（域名池/IP 池）
- 网络隔离（VLAN Tag）
- 策略隔离（独立防御策略）

**动态防火墙**：
- 1 分钟一次健康检查
- 检测 DDoS（流量超阈值）
- 自动熔断（隔离受攻击蜂窝）

### 死因分析服务

**分析流程**：
1. 收集特征（丢包率/RTT/DNS 响应时间）
2. ML 分类（DNS 污染/IP 封锁/证书吊销）
3. 生成报告（死因/存活时间/触发特征）
4. 优化建议（避开注册商/提高轮换频率）

**特征提取**：
- 丢包率
- RTT 方差
- ICMP 不可达次数
- DNS 响应时间
- 注册商
- 顶级域名

### 策略引擎服务

**策略计算**：
1. 环境感知（地理位置/时间/网络类型）
2. 威胁评估（威胁等级）
3. 策略选择（拟态模板/域名池/填充比例）

**填充比例计算**：
- 基础填充：根据威胁等级（10%-30%）
- 用户偏好调整：
  - 经济模式：减半
  - 平衡模式：默认
  - 极限模式：1.5 倍

**热下发策略**：
1. 查找用户所在蜂窝
2. 推送到该蜂窝的所有 Gateway
3. Gateway 立即应用新策略

### 威胁情报聚合服务

**聚合流程**：
1. 存储原始报告
2. 更新黑名单（高严重度威胁）
3. 触发自动响应（主动探测 → 紧急切换）
4. 生成情报摘要

**黑名单分发**：
- 加密分发到所有 Gateway
- 全局黑名单同步
- 秒级生效

## 核心功能模块

### 用户蜂窝管理

**蜂窝定义**：
- ID / 名称 / 区域
- 安全等级
- 最大域名数 / 最大用户数
- 域名列表 / 用户列表
- 健康评分

**蜂窝操作**：
- 创建蜂窝
- 分配用户到蜂窝
- 蜂窝隔离
- 动态防火墙

### 成本计算器

**月流量成本计算**：
- 基础流量 × $0.10/GB
- 额外流量（Padding）× $0.05/GB
- 总成本 = 基础成本 + 额外成本

**实时成本预估**：
- 用户历史流量
- 当前填充比例
- 基础成本 / 额外成本 / 总成本
- 节省空间

## gRPC 核心逻辑

### SyncHeartbeat（生死裁决）

**功能**：
- 更新 Gateway 在线状态
- 流量结算（原子扣减配额）
- 计费流水记录
- 熔断决策（配额 ≤ 0 时返回 remaining_quota=0）
- 动态防御配置

### ReportTraffic（流量上报）

**功能**：
- 10 秒一次流量增量上报
- 计算费用（业务流量 + 防御流量）
- 应用蜂窝成本倍率
- 原子扣减配额
- 配额告警

### ReportThreat（威胁上报）

**功能**：
- 实时威胁情报记录
- 累加命中次数
- 全局威胁决策（命中 ≥ 100 次触发全局封禁）

### GetQuota（配额查询）

**功能**：
- 查询用户剩余配额
- 返回配额信息

### getGlobalBlacklist（全局黑名单分发）

**功能**：
- 查询最近 1 小时 Top 100 威胁 IP
- 分发到所有 Gateway
- 内核态 LPM Trie Map 更新

### 审计日志

**记录内容**：
- 时间戳
- 操作类型
- 用户 ID

## 总结

Mirage-OS 是一个集成了全球域名流水线调度、蜂窝隔离算法、死因分析算法、Web 指挥中心的控制中心，通过高性能微服务架构和前端看板，实现了：

- **全球域名流水线调度**：自动采购、温储备池、司法管辖区轮转、G-Switch 扩散、死信开关
- **蜂窝隔离算法**：用户分组隔离、资源池独立、风险对冲、动态防火墙、洁净区溢价
- **死因分析算法**：封禁特征识别、存活周期建模、对抗策略评估、自动优化建议、情报库进化
- **Web 指挥中心**：蜂窝可视化仪表盘、策略热下发、威胁情报中心、死因分析报告、用户管理

通过 gRPC 生死裁决、流量结算、威胁上报、全局黑名单分发，实现了全球蜂窝网络的统一调度和管理。通过 WebSocket 实时推送，实现了前端态势感知的实时更新。

这是一个自进化、自防御、自计费的去中心化网络控制中心。
