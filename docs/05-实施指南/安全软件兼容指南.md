# Mirage 安全软件兼容指南

## 概述

Mirage Gateway 使用 eBPF 技术进行网络流量处理，某些安全软件可能将其标记为可疑行为。本指南说明如何正确配置以避免误报。

---

## 为什么需要特殊权限

| 权限 | 用途 | 说明 |
|------|------|------|
| CAP_BPF | 加载 eBPF 程序 | 流量伪装和指纹修改的核心功能 |
| CAP_NET_ADMIN | 网络配置 | 挂载 XDP/TC 程序到网卡 |
| CAP_SYS_ADMIN | 系统管理 | 访问内核数据结构 |
| root | 完整权限 | eBPF 程序加载的最低要求 |

---

## 正常网络行为说明

Mirage Gateway 运行时会产生以下网络行为，均为正常现象：

1. **周期性心跳** - 每 30 秒向控制中心发送状态报告
2. **流量填充** - 在业务流量中添加随机填充数据
3. **噪声注入** - 生成模拟正常网络活动的背景流量
4. **多路径传输** - 同时使用多条网络路径传输数据

## Windows Defender 配置

如果使用 Windows 版本的 mirage-cli：

```powershell
# 以管理员身份运行 PowerShell
Add-MpPreference -ExclusionPath "C:\Program Files\Mirage"
Add-MpPreference -ExclusionProcess "mirage-cli.exe"
```

## Linux 安全模块配置

### SELinux

```bash
# 创建自定义策略模块
cat > mirage-gateway.te << 'EOF'
module mirage-gateway 1.0;

require {
    type unconfined_t;
    class bpf { map_create prog_load };
}

allow unconfined_t self:bpf { map_create prog_load };
EOF

# 编译并安装
checkmodule -M -m -o mirage-gateway.mod mirage-gateway.te
semodule_package -o mirage-gateway.pp -m mirage-gateway.mod
semodule -i mirage-gateway.pp
```

### AppArmor

```bash
# /etc/apparmor.d/opt.mirage-gateway.bin.mirage-gateway
#include <tunables/global>

/opt/mirage-gateway/bin/mirage-gateway {
  #include <abstractions/base>
  #include <abstractions/nameservice>
  
  capability net_admin,
  capability sys_admin,
  capability bpf,
  
  network inet stream,
  network inet dgram,
  
  /opt/mirage-gateway/** r,
  /etc/mirage/** r,
  /var/log/mirage/** rw,
  /sys/fs/bpf/** rw,
}
```

## 企业 EDR 白名单申请

如果您的组织使用企业级 EDR（如 CrowdStrike、SentinelOne），请联系 IT 部门提交白名单申请，提供以下信息：

1. **程序路径**: `/opt/mirage-gateway/bin/mirage-gateway`
2. **程序哈希**: 运行 `sha256sum /opt/mirage-gateway/bin/mirage-gateway`
3. **签名证书**: 提供代码签名证书信息
4. **行为说明**: 引用本文档的"正常网络行为说明"部分

## 验证安装

运行以下命令验证 Mirage Gateway 正常工作：

```bash
# 检查服务状态
systemctl status mirage-gateway

# 检查 eBPF 程序加载
bpftool prog list | grep mirage

# 检查网络连接
ss -tlnp | grep mirage
```

## 故障排除

### 问题：服务启动失败，提示权限不足

```bash
# 检查 capabilities
getcap /opt/mirage-gateway/bin/mirage-gateway

# 设置 capabilities（如果未使用 systemd）
setcap cap_bpf,cap_net_admin,cap_sys_admin+ep /opt/mirage-gateway/bin/mirage-gateway
```

### 问题：安全软件持续告警

1. 确认已添加白名单规则
2. 检查程序签名是否有效
3. 联系安全软件厂商提交误报
