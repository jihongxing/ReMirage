\# Mirage-OS: 零信任抗审计与防守侧归因架构

\*\*版本:\*\* v1.0.0

\*\*核心原则:\*\* Behavior Over Identity (行为优于身份) \& Default-Drop (默认静默丢弃)



\## 1. 架构哲学：防守侧归因



在应对国家级防火墙（GFW）、云厂商合规扫描以及高级持续性威胁（APT）的主动探测时，传统的“维护恶意 IP 黑名单”方案已被证明是脆弱且低效的。攻击者可以通过 BGP 劫持、僵尸网络或动态住宅 IP 轻易绕过身份封锁。



Mirage 系统彻底抛弃了“主动定位审计者基础设施”的猫鼠游戏，全面转向\*\*基于系统自身遥测数据的“防守侧行为归因”\*\*。



我们不关心来访者的 IP 是什么，我们只通过其接触网关时的\*\*微观动作、时序特征和上下文\*\*来判定其合法性。任何不严格遵守 Mirage 交互规则的流量，都将在物理空间被抹除。



\## 2. 威胁情报与上下文富化 (Defensive Enrichment)



系统不依赖外部的“黑名单”，而是使用通用公开数据（OSINT）对入站流量进行\*\*风险评分（Risk Scoring）富化\*\*。



\### 2.1 引入的数据源

\* \*\*云厂商公开网段 (Cloud Ranges):\*\* AWS, Azure, GCP, Aliyun, Tencent Cloud 等官方公布的 IP ranges。

\* \*\*ASN / WHOIS 库:\*\* 离线的高性能 IP-to-ASN 数据库。

\* \*\*RPKI (Resource Public Key Infrastructure):\*\* 全球路由源认证数据。

\* \*\*互联网背景噪音 (Internet Noise):\*\* 类似 GreyNoise 提供的大规模无差别扫描器特征库。



\### 2.2 富化逻辑

以上数据不作为“直接阻断”的依据，而是作为异常上下文的判断标准。例如：

\* 合法的高净值流量通常来自家庭宽带（ISP ASN）或 5G 移动网络。

\* 如果携带合法证书的流量来自“阿里云杭州数据中心”，该连接将被标记为高风险异常，强制触发重认证或降级处理。

\* 触发异常的 IP 若在背景噪音库中命中（如 Shodan 爬虫），则不触发高级安全告警，以防告警疲劳。



\## 3. 三层纵深防御矩阵 (Defense-in-Depth)



系统采用“滴水不漏”的 Default-Drop 策略，将防御分为内核、密码学和行为三个层级。



\### L1: 物理与链路层清洗 (eBPF XDP/TC)

利用 eBPF 在网卡驱动层实现极低延迟的早期过滤，保护应用层免受资源枯竭攻击。

\* \*\*静态 ASN/网段清洗:\*\* 明确配置的敌对 ASN 或云厂商数据中心 IP，在匹配后直接执行 `XDP\_DROP`。

\* \*\*速率限制 (Rate Limiting):\*\* 防御 SYN Flood 或高频连接尝试。

\* \*\*静默响应:\*\* 拒绝产生任何 ICMP Unreachable 回包，让攻击者的扫描器陷入等待超时的黑洞。



\### L2: 密码学与协议防线 (User-Space)

防御针对协议栈和密码学原语的主动探测。

\* \*\*严格 mTLS 校验:\*\* 必须出示由系统私有 CA 签发的 Client Certificate。

\* \*\*抗重放归因 (Anti-Replay):\*\* 记录握手阶段的 Nonce 与 Timestamp。如果 DPI 截获了合法握手包并异地重放探测，系统将识别出重复的密码学特征，并立即封锁该特征关联的所有连接。

\* \*\*半开连接/慢速嗅探熔断:\*\* 任何未能在设定阈值（如 300ms）内完成完整安全握手的连接，直接在 TCP/UDP 层静默 RST，绝不抛出应用层错误码（防范框架指纹泄漏）。



\### L3: 统计与行为基线监控 (Behavioral Baseline)

针对已经建立连接的流量进行持续性行为画像。

\* \*\*非预期协议突发归因:\*\* 若 443 端口收到 SSH 握手、HTTP 明文请求等探测报文，立即标记来源 IP 为“协议扫描器”。

\* \*\*时序与熵值校验:\*\* 监控连接的包长分布与收发比例。若流量特征严重偏离 Mirage 内置的马尔可夫链模型（如只发不收、或者表现出明显的自动化扫描特征），网关在下一心跳周期强制踢下线。



\## 4. 实施规范与最佳实践



1\.  \*\*数据本地化闭环:\*\* 所有富化数据（ASN 库、云厂商 IP 库）必须通过安全通道定期推送到网关内存中，防御判定必须在本地完成，\*\*绝对禁止\*\*在数据面处理连接时向外发起同步的网络查询（防止被时序侧信道攻击或 DDoS 拖垮）。

2\.  \*\*告警静默:\*\* 网关对非法请求的唯一正确反应是\*\*静默丢弃\*\*。禁止在对外的响应中包含任何可以识别 Mirage 系统的错误堆栈、特定的 HTTP 状态码或特殊的 TLS Alert 报文。

3\.  \*\*零信任准入:\*\* 永远不要因为某个 IP 昨天是合法的，今天就放松对它的 L2/L3 校验。
