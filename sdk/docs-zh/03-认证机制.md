# 认证机制

## 概述

Mirage 使用密钥对认证，无需用户名密码，完全匿名。

认证流程：
1. 生成密钥对（secp256k1）
2. 注册公钥获取账户
3. 私钥签名挑战换取 Token
4. Token 用于后续请求

---

## 密钥对生成

### SDK 方式

```python
from mirage import MirageClient

# 生成密钥对
keypair = MirageClient.generate_keypair()

# 保存（私钥 AES-256 加密）
keypair.save("~/.mirage/keys", password="optional_password")

# 加载
keypair = MirageClient.load_keypair("~/.mirage/keys", password="optional_password")
```

### 命令行方式

```bash
# 生成
mirage-cli keygen --output ~/.mirage/keys

# 带密码保护
mirage-cli keygen --output ~/.mirage/keys --password

# 查看公钥
mirage-cli pubkey --keyfile ~/.mirage/keys
```

### 手动方式（OpenSSL）

```bash
# 生成私钥
openssl ecparam -genkey -name secp256k1 -out private.pem

# 导出公钥
openssl ec -in private.pem -pubout -out public.pem
```

## 账户注册

首次使用需注册公钥：

```python
from mirage import MirageClient

# 匿名连接
client = MirageClient.anonymous(endpoint="grpc.mirage.example:50847")

# 注册
account = client.register(public_key=keypair.public_key)

# 保存账户信息
account.save("~/.mirage/account")
```

注册响应：

```json
{
  "account_id": "acc-a1b2c3d4",
  "created_at": 1704067200,
  "deposit_address": "4xxx...xxx"
}
```

## 签名认证

### 认证流程

```
客户端                              服务端
   |                                   |
   |  1. 请求挑战 (account_id)          |
   | --------------------------------> |
   |                                   |
   |  2. 返回挑战 (nonce + timestamp)   |
   | <-------------------------------- |
   |                                   |
   |  3. 私钥签名挑战                    |
   | --------------------------------> |
   |                                   |
   |  4. 验证签名，返回 JWT Token        |
   | <-------------------------------- |
```

### SDK 自动处理

```python
# 自动完成签名认证
client = MirageClient.connect(
    endpoint="grpc.mirage.example:50847",
    keyfile="~/.mirage/keys"
)
# Token 自动获取和刷新
```

### 手动认证

```python
from mirage import MirageClient

# 1. 请求挑战
client = MirageClient.anonymous(endpoint="grpc.mirage.example:50847")
challenge = client.auth.request_challenge(account_id="acc-a1b2c3d4")

# 2. 签名挑战
signature = keypair.sign(challenge.nonce + challenge.timestamp)

# 3. 验证获取 Token
token_response = client.auth.verify_signature(
    account_id="acc-a1b2c3d4",
    signature=signature
)

# 4. 使用 Token
client = MirageClient(
    endpoint="grpc.mirage.example:50847",
    token=token_response.token
)
```

## JWT Token

### Token 结构

```json
{
  "sub": "acc-a1b2c3d4",
  "iat": 1704067200,
  "exp": 1704153600,
  "scope": ["gateway", "billing", "cell"]
}
```

### Token 有效期

- 默认有效期：24 小时
- SDK 自动刷新：过期前 1 小时自动续期
- 手动刷新：调用 `client.auth.refresh_token()`

### Token 使用

所有 API 请求自动携带：

```
Authorization: Bearer <token>
```

## gRPC 认证

### 拦截器方式

```go
// Go
func authInterceptor(token string) grpc.UnaryClientInterceptor {
    return func(ctx context.Context, method string, req, reply interface{},
        cc *grpc.ClientConn, invoker grpc.UnaryInvoker, opts ...grpc.CallOption) error {
        ctx = metadata.AppendToOutgoingContext(ctx, "authorization", "Bearer "+token)
        return invoker(ctx, method, req, reply, cc, opts...)
    }
}
```

```python
# Python
class AuthInterceptor(grpc.UnaryUnaryClientInterceptor):
    def __init__(self, token):
        self.token = token
    
    def intercept_unary_unary(self, continuation, client_call_details, request):
        metadata = [("authorization", f"Bearer {self.token}")]
        new_details = client_call_details._replace(metadata=metadata)
        return continuation(new_details, request)
```

## WebSocket 认证

### 连接时认证

```javascript
const ws = new WebSocket('wss://ws.mirage.example:18443');

ws.onopen = () => {
  ws.send(JSON.stringify({
    type: 'auth',
    token: 'your_jwt_token'
  }));
};
```

### 认证响应

```json
// 成功
{ "type": "auth_success", "expires_at": 1704153600 }

// 失败
{ "type": "auth_failed", "error": "invalid_token" }
```

## 权限范围

| Scope | 权限 |
|-------|------|
| gateway | 心跳/流量/威胁上报 |
| billing | 余额/充值/购买 |
| cell | 蜂窝查询/分配/切换 |
| admin | 管理员操作 |

---

## 安全建议

1. Token 存储在内存，不持久化
2. 使用 TLS 加密传输
3. 定期轮换 Token
4. 监控异常登录
